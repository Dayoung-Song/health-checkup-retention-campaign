# RPA 기획

## 자동화 목적

수기 운영에서 반복적으로 수행하던 대상자 추출, 발송 파일 생성, 결과 집계 과정을 자동화하여 운영 효율과 처리 안정성 강화

## RPA 범위

| 기능 | 설명 |
|:---:|:---|
| 기준일 계산 | 발송일 기준 45주 전 계산 |
| 후보군 추출 | 발송일 기준 45주 전(약 10개월 전) 대상자 추출 |
| 제외 조건 적용 | 무효 예약번호, 최근 예약, 연락처 수신 불가, 중복 대상 제외 |
| 발송 파일 생성 | 외부 발송 채널에 전달 가능한 형식으로 파일 생성 |
| 예약 결과 매칭 | 발송 수신자와 예약 데이터를 연결 |
| 성과 집계 | 측정 기간 내 예약률 계산 |

## 요구사항 요약

- 기준일 자동 계산 및 입력
- 기준일을 입력하면 발송 후보군 자동 생성
- 발송 이후 발송 시도 건수 확인 가능
- 최종 발송 대상자 분리 저장
- 발송 결과와 예약 결과를 매칭 가능
- 오류 발생 시 원인을 확인할 수 있는 로그가 기록
- 개인정보가 포함된 파일은 접근 권한 제한
- 원하는 시점(요일, 시간대) 예약 발송

## 협업 방식

- 서비스기획: 업무 프로세스, 데이터 처리 조건, 성과 지표 정의 및 확인
- 개발 부서: RPA 구현, 실행 환경 구성, 오류 처리

## 인수 기준

| 항목 | 기준 |
|---|---|
| 대상자 추출 정확도 | 수기 기준과 비교해 주요 조건이 동일하게 반영됨 |
| 파일 생성 | 발송 요청 형식에 맞는 파일이 자동 생성됨 |
| 로그 | 실행 결과와 오류 사유가 기록됨 |
| 재실행 | 오류 수정 후 동일 기준일로 재처리 가능 |
| 성과 집계 | 발송 후 예약률을 산출할 수 있음 |
