# 문제 정의

## 배경

- 건강검진 서비스 도메인은 정기적 재방문 가능성은 높지만, 성수기/비수기 구분이 뚜렷함
- 최근 건강검진 서비스를 제공하는 병원이 많으므로, 반드시 같은 기관에 방문하리라는 보장 없음
- 2년에 1회 제공되는 국가건강검진 특성 상 재방문까지 장기간 소요됨

## 문제

- 재방문 가능성이 있는 수검자에게 적절한 시점의 알림이 제공되지 않음
- 알림 메세지 대상자 선별과 알림 메세지 요청이 엑셀 수작업 중심으로 운영
- 알림 메세지 발송 가능한 수검자와 제외해야 하는 수검자를 구분하는 기준이 명확히 문서화되어 있지 않음
- 알림 메세지 발송 후 예약 전환 성과를 정기적으로 확인하기 어려움

## 기획 목표

- 검진일 기준 일정 기간이 지난 고객 중 알림 메세지 발송 가능한 대상자를 추출
- 최근 예약 여부, 중복 여부를 고려해 제외 조건 정의
- 수기 운영으로 성과 먼저 검증
- 검증된 운영 프로세스를 RPA로 자동화
- 자동화 후 예약률을 재측정해 개선 효과 확인

## 성공 지표

| 지표 | 설명 |
|---|---|
| 발송 가능 대상자 수 | 제외 조건을 통과한 최종 발송 대상자 수 |
| 예약률 | 발송 수신자 중 측정 기간 내 예약한 비율 |
| 운영 소요 시간 | 대상자 추출부터 결과 집계까지 걸린 시간 |
| 자동화 처리 안정성 | RPA 처리 중 오류 또는 누락 발생 여부 |
