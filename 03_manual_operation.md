# 수기 운영

## 운영 목적

RPA 개발 전, 수기 운영을 통해 발송 대상자 추출 기준과 성과 측정 방식 검증

## 운영 프로세스

1. 기준월에 맞춰 후보 데이터 추출
2. 발송 제외 조건을 적용해 최종 대상자 생성
3. 외부 메시지 발송 채널에 발송 요청 파일 전달
4. 발송 완료 후 수신자 기준으로 예약 여부 확인
5. 발송 후 7일간 예약률 측정
6. 성과를 정리해 자동화 필요성 검토

## 수기 운영에서 확인한 문제

- 매회 동일한 전처리 작업을 반복
- 대상자 추출 기준이 사람 확인에 의존
- 발송 요청 파일 생성과 결과 확인에 시간 소요
- 예약 결과 매칭 과정에서 누락 또는 중복 확인이 필요
- 월 단위 발송에 의한 상담 요청 과부하(단기간 상담 요청 쏠림 현상)

## 수기 운영 성과

| 항목 | 결과 |
|---|---:|
| 운영 기간 | 9개월 |
| 측정 기준 | 발송 후 7일 예약률 |
| 예약률 | 2.9% |
| 매출 효과 | 비식별 기준으로 유의미한 매출 기여 확인 |

## 자동화 전환 판단

수기 운영을 통해 발송 대상자 추출 기준과 예약률 측정 방식 검증 완료되었으며,  반복 작업 비중이 높아 RPA 자동화 대상으로 적합하다고 판단
