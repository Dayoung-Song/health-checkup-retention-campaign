# 대상자 추출 로직

## 핵심 원칙

대상자 추출은 고객 특성별 세분화보다, 먼저 알림 메세지를 발송해도 괜찮은 대상자를 안전하게 추출하는 데 초점

## 데이터 처리 사용 툴
Python

## 추출 기준

| 구분 | 조건 | 처리 |
|---|---|---|
| 검진월 기준 | 최근 검진 후 약 300일 경과 수검자 월단위 추출 | 후보군 포함 |
| 예약번호 유효성 | 일부 연기된 검진을 진행한 예약번호 | 제외 |
| 최근 예약 여부 | 이미 예약했거나, 300일 전 검진 이후에 재방문한 수검자 | 제외 |
| 연락처 유효성 | 연락처 누락 또는 형식 오류 | 제외 |
| 연락처 중복 여부 | 동일 연락처 | 제외 |
| 수검자 중복 여부 | 동일 수검자 중복 | 최신 검진일 예약번호 1건만 잔류 |
| 발송 제한 | 내부 정책상 발송 제외 대상 | 제외 |

## 처리 흐름

```text
전체 후보군
→ 검진월 기준 약 300일 경과 수검자 추출(월단위)
→ 연락처 유효성 확인
→ 예약번호 유효성 확인
→ 발송 제한 조건 적용
→ 최근 예약자 제외
→ 수검자 중복 대표 1건만 잔류
→ 연락처 중복 제외
→ 최종 발송 대상자 생성
```

## 산출물

- 최종 발송 대상자 리스트
- 발송 요청용 파일
- 발송 후 예약 결과 매칭용 기준 파일

## 주의 사항

- 실제 고객명, 전화번호, 생년월일 등 직접 식별자는 공개 저장소에 포함하지 않음
- 회사 내부의 상세 제외 조건은 공개하지 않고, 일반화된 조건으로 설명
- 발송 문구 원본은 공개하지 않고, 메시지 목적과 구조만 설명
